##Outage / Activity Details This MCM is in response to a recordable safety incident that happened from a SmartPac 5.3 machine at AUS2. An AA reached into the machine to remove a P-Slip from blocking a sensor. Once the P-Slip was removed, there was movement within the machine that created impact to the AA's fingers. The SmartPac machine is designed to allow the gripper component inside of the machine to move up, down, and close even while the light curtain is broken. This has been justified in the design since deployment of the machines due to the low force the gripper exerts and guidance from applicable biomechanical limits standards. The low risk of this component led to administrative controls as an appropriate risk reduction measure. With this recordable injury, the Pack team and ATS (SmartPac Manufacturer)revisited the risk of the gripper component and worked on how to further mitigate any risk of impact. The Pack team has created a code to change the PLC to eliminate the action of gripper closing and the gripper upstroke movement when the light curtain is broken. This has been tested at both AUS2 and TUS2 on the SmartPac 5.3 machines as well as SmartPac 3.x machines. #### What is the purpose of this activity or change? To remove the risk of an additional safety incident by adding a code change to the PLC for the gripper component’s close and upward stroke motion to correspond with the light curtain. Plan • Download the revised PLC code into one SmartPac machine per Amazon site (both 3 and 5 versions) • Have it tested for a shift to address potential issues • If problems are experienced on this machine, report back to Pack Auto for additional support • Download the revised PLC code into the remaining SmartPac machines per Amazon site #### What will be required to execute this change? - RME assistance on putting the provided code into the PLC #### What is the expected end state of the system after this change? The machine cycle time will be unchanged, but the sequence of the machine will be different. This will require AAs to adjust their muscle memory which may require a ramp up period. #### What assumptions, if any, are being made about the state of the system at the time of this change? Increased SmartPac safety ##Impact / Risk Assessment #### Why is it necessary? What is the impact of not making this change? This is to address a recordable incident from the gripper component. This change is to avoid seeing this again in the future. #### Why does this activity or change need to be done under MCM? Can it be safely automated? This requires machine PLC changes Code versions required (repository link provided in "Related Items": SmartPac 3 all versions: SmartPac3_v32_Uni3Bag_2022_02_28_1800.ACD HMI needed: SmartPac3Hmi_v11_2022_02_27_r10.mer SmartPac 5 (repository link provided in "Related Items"): Replaces versions dated “2021_08_25_1100” -> SmartPac5_v32_GrpFix10_2021_08_25_1100.ACD SmartPac5_v32_AC73bB11_2022_02_27_1800.ACD HMI needed for either of the above versions: SmartPac5Hmi_v11_2022_02_25_r10.mer #### Why is this the correct time/day to complete the CM)?
